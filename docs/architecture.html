mcp-server-database

MCP server for relational database operations across PostgreSQL, MySQL, and SQLite

TypeScript 5.3+ MCP Server v1.0.0

Executive Summary

An MCP (Model Context Protocol) server that provides AI assistants with tools for interacting with relational databases. Supports PostgreSQL, MySQL, and SQLite through a unified interface. The server exposes six tools for querying, schema introspection, table listing, database discovery, migration execution, and full DDL export. Built with the official MCP SDK and driver-specific Node.js clients (pg, mysql2, better-sqlite3), it communicates via stdio transport for seamless integration with Claude Desktop and other MCP clients.

Data Flow / Request Flow

  1. Client Request -- An MCP client sends a JSON-RPC tool invocation over stdio with the tool name and parameters.
  2. Parameter Validation -- The server validates parameters using Zod schemas specific to each tool.
  3. Driver Resolution -- The connection manager selects the appropriate database driver based on the DB_DRIVER environment variable.
  4. Query Execution -- The tool handler executes the database operation using parameterized queries to prevent SQL injection.
  5. Result Serialization -- Query results are serialized as structured JSON and returned to the MCP client.

Production Readiness Checklist

  • Configure database driver and connection credentials via environment variables
  • Create a dedicated database user with minimum required permissions
  • Enable TLS/SSL for PostgreSQL and MySQL connections
  • Build TypeScript to JavaScript (npm run build)
  • Test all six tools against target database
  • Configure MCP client with server entry point and database environment
  • Set up database audit logging for query tracking
  • Deploy as Docker container for production use

Deployment

Local Development

git clone https://github.com/kogunlowo123/mcp-server-database.git
cd mcp-server-database
npm install
npm run build
DB_DRIVER=postgres DB_HOST=localhost \
  DB_USER=admin DB_PASSWORD=secret \
  DB_NAME=mydb npm start

Docker / Production

docker build -t mcp-server-database .
docker run \
  -e DB_DRIVER=postgres \
  -e DB_HOST=host.docker.internal \
  -e DB_USER=admin \
  -e DB_PASSWORD=secret \
  -e DB_NAME=mydb \
  mcp-server-database
